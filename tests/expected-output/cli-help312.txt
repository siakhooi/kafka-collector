usage: kafka-collector [-h] [-v] -t TOPICS [-b BOOTSTRAP_SERVER] [-g GROUP]
                       [-o OUTPUT] [-c CAPTURE_DIR] [-m {cli,service}]

collect kafka messages from multiple topics

options:
  -h, --help            show this help message and exit
  -v, --version         show program's version number and exit
  -t TOPICS, --topics TOPICS
                        comma separated list of kafka topics to be listened to
  -b BOOTSTRAP_SERVER, --bootstrap-server BOOTSTRAP_SERVER
                        kafka bootstrap server (default: localhost:9092)
  -g GROUP, --group GROUP
                        consumer group id (default: random uuid)
  -o OUTPUT, --output OUTPUT
                        output file path, use '-' for stdout (default: stdout)
  -c CAPTURE_DIR, --capture-dir CAPTURE_DIR
                        capture directory for service mode (default:
                        /tmp/kafka-collector)
  -m {cli,service}, --mode {cli,service}
                        run mode: cli or service (default: cli)
